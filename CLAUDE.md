# NMA Workspace — CLAUDE.md

## Project Overview

This workspace is the **Neuromatch Academy (NMA)** educational content and CI/CD infrastructure hub. It contains:

- **`nmaci/`** — The central CI/CD automation tool used by all course repos
- **`course-content/`** — Computational Neuroscience course (compneuro.neuromatch.io)
- **`neuro-ai-course-content/`** — NeuroAI course (neuroai.neuromatch.io)
- **`course-content-dl/`** — Deep Learning course (deeplearning.neuromatch.io)
- **`climate-course-content/`** — ClimateMatch course (comptools.climatematch.io)

All repos are independent git repositories under a shared workspace managed by `pixi.toml`.

## Architecture

### nmaci (CI/CD Tool)

`nmaci` is a pip-installable Python package (`pip install git+https://github.com/OleBialas/nmaci@main`). All logic lives in `src/nmaci/` and is accessible via the `nmaci` CLI:

| CLI command | Module | Purpose |
|---|---|---|
| `nmaci process-notebooks` | `process_notebooks.py` | Executes notebooks, extracts solutions, creates student/instructor versions |
| `nmaci verify-exercises` | `verify_exercises.py` | Checks exercise cells match solution cells (allows `...` and comments) |
| `nmaci lint-tutorial` | `lint_tutorial.py` | Runs flake8/pyflakes over notebook code cells |
| `nmaci generate-readmes` | `generate_tutorial_readmes.py` | Auto-generates `README.md` with Colab/video/slide links |
| `nmaci generate-book` | `generate_book.py` | Builds Jupyter Book from `materials.yml` |
| `nmaci generate-book-dl` | `generate_book_dl.py` | Builds Jupyter Book (Deep Learning variant) |
| `nmaci generate-book-precourse` | `generate_book_precourse.py` | Builds Jupyter Book (Precourse variant) |
| `nmaci select-notebooks` | `select_notebooks.py` | Filters which notebooks to process (excludes student/instructor copies) |
| `nmaci make-pr-comment` | `make_pr_comment.py` | Creates PR comment with Colab badges and lint report |
| `nmaci find-unreferenced` | `find_unreferenced_content.py` | Identifies unused solution images/scripts |
| `nmaci extract-links` | `extract_links.py` | Extracts video/slide links from notebooks |
| `nmaci parse-html` | `parse_html_for_errors.py` | Checks HTML build output for errors |

**Repo structure:**
```
nmaci/
  pyproject.toml               # package manifest (hatchling, uv)
  src/nmaci/                   # source of truth — edit these
    cli.py                     # subcommand dispatcher
    *.py                       # one module per command
    chatify/                   # experimental chatify subpackage
  scripts/                     # thin shims for backward compat (do not edit)
  tests/
    test_process_notebooks.py
    tutorials/                 # fixture notebooks
```

`scripts/` contains shims that delegate to `src/nmaci/` — they exist for backward compat with course repo CI that still downloads and runs `python ci/script.py`. They will be removed in Phase 2.

### Course Repos

Each course follows an identical structure:

```
tutorials/
  materials.yml               # course metadata: video links, slides, day names
  W{week}D{day}_{TopicName}/
    W{week}D{day}_Intro.ipynb
    W{week}D{day}_Tutorial{N}.ipynb   # source of truth — edited by humans
    W{week}D{day}_Outro.ipynb
    README.md                         # auto-generated, do not edit manually
    student/                          # auto-generated by nmaci
    instructor/                       # auto-generated by nmaci
    solutions/                        # auto-generated .py files (MD5-hashed names)
    static/                           # auto-generated solution images
projects/
book/
  _config.yml                         # Jupyter Book config
environment.yml
requirements.txt
.github/workflows/
  notebook-pr.yaml            # main CI: triggered on PR with .ipynb changes
```

## Key Conventions

### Notebook Versions

The **source notebooks** (at `W{week}D{day}_{TopicName}/W{week}D{day}_Tutorial{N}.ipynb`) are the **only files humans should edit**. The following are always auto-generated and should never be manually edited:

- `student/` — solution cells replaced with images + `...`
- `instructor/` — code moved to markdown
- `solutions/` — extracted solution `.py` files
- `static/` — rendered solution images (`.png`)
- `README.md` per tutorial directory

### Exercise/Solution Pattern

- Solution cells are marked with cell tags in the notebook metadata
- `verify_exercises.py` checks alignment between exercise stubs and solutions
- Exercise stubs use `...` (ellipsis) or commented-out code as placeholders
- `NotImplementedError` in solutions is allowed and expected during execution

### materials.yml

This file in each course's `tutorials/` directory drives:
- Book table of contents generation
- README link generation
- PR comment badge generation

Fields per day entry: `day_code`, `category`, `name`, `youtube`, `bilibili`, `slides`, `tutorial_count`, etc.

### Solution File Naming

Solution scripts use MD5 hashes in filenames (e.g., `W1D1_Tutorial1_Solution_abc123de.py`) — this is intentional to avoid collisions. Do not rename them.

### Branch-Aware CI

Course repo workflows support testing against a specific `nmaci` branch: include `nmaci:branch-name` in the commit message to override which branch is used. Currently course repos download a tarball and run scripts from `ci/`; after Phase 2 they will `pip install git+https://github.com/OleBialas/nmaci@<branch>` instead.

## Development Workflow

### Making Changes to nmaci

1. Edit modules in `nmaci/src/nmaci/` (never `scripts/` — those are shims)
2. Install locally: `cd nmaci && uv pip install -e ".[dev]"`
3. Run tests: `pytest tests/ src/nmaci/`
4. Push to a branch; course repos can test against it using `nmaci:your-branch` in commit messages

### Making Changes to Course Notebooks

1. Edit only the **source notebooks** (never `student/`, `instructor/`, `solutions/`, or `README.md`)
2. Open a PR — GitHub Actions will automatically process notebooks and push the generated files to the PR branch
3. CI runs: execute → verify exercises → lint → generate READMEs → PR comment with Colab badges

### Skipping CI

Include `skip ci` in the commit message to bypass notebook processing in GitHub Actions.
